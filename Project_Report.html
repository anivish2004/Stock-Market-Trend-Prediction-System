Abstract

Stock market prediction is one of the most challenging tasks in the domain of financial analytics due to the inherently volatile and non-linear nature of market data. This project presents a Stock Market Trend Prediction System that employs a Hybrid Artificial Neural Network–Genetic Algorithm (ANN-GA) approach to forecast short-term stock price direction—classifying the next trading session as Bullish, Bearish, or Neutral.

The system fetches historical OHLCV (Open, High, Low, Close, Volume) data from Yahoo Finance using the yfinance library, engineers a comprehensive set of technical indicators—including Simple Moving Average (SMA), Exponential Moving Average (EMA), Relative Strength Index (RSI), Moving Average Convergence Divergence (MACD), momentum, and volatility metrics—and constructs a feature matrix for model training. A Genetic Algorithm is employed to perform automated feature selection and hyperparameter tuning for a Multi-Layer Perceptron (MLP) neural network classifier, thereby optimizing prediction accuracy without manual intervention.

The front-end application is built using Streamlit, offering a professional, chart-first trading-terminal interface with interactive Plotly visualizations including candlestick charts, RSI plots, MACD histograms, probability gauges, and confusion matrix heatmaps. The system also integrates with MongoDB Atlas for persistent storage of prediction logs, enabling users to search, audit, and compare historical runs over time.

The project demonstrates that evolutionary optimization combined with neural network classification can produce reliable decision-support signals for stock market analysis, achieving competitive accuracy on validation data across multiple ticker symbols from both US and Indian (NSE) markets.

Keywords: Stock Market Prediction, Artificial Neural Network, Genetic Algorithm, Technical Indicators, Feature Selection, Streamlit, MongoDB, Machine Learning, Financial Analytics.

Chapter 1: Introduction

1.1 Overview

Financial markets generate enormous volumes of price and trading data every day. Predicting the future direction of stock prices has long been a goal for traders, analysts, and researchers alike. Traditional approaches such as fundamental analysis rely on balance sheets and earnings reports, while technical analysis leverages historical price patterns and indicators. With the advent of machine learning, computational methods have emerged as a powerful complement to both.

This project, titled "MarketPulse AI – Stock Market Trend Prediction System," aims to build an intelligent decision-support tool that predicts whether the next trading session for a given stock will likely move upward (Bullish), downward (Bearish), or remain relatively flat (Neutral). The system uses a hybrid approach combining an Artificial Neural Network (ANN) for classification and a Genetic Algorithm (GA) for automated feature selection and hyperparameter optimization.

1.2 Background

Stock markets are inherently non-linear, noisy, and influenced by a complex interplay of economic indicators, market sentiment, geopolitical events, and investor psychology. Traditional linear models such as ARIMA have limited capacity to capture these complexities. Neural networks, particularly Multi-Layer Perceptrons (MLPs), can model non-linear decision boundaries, making them suitable for binary classification tasks such as predicting market direction.

However, the performance of an ANN depends heavily on the choice of input features and hyperparameters. A Genetic Algorithm—an evolutionary optimization technique inspired by natural selection—is used here to search across a space of feature subsets and network configurations, selecting the combination that maximizes prediction accuracy on held-out validation data.

1.3 Motivation

The motivation behind this project is to demonstrate how bio-inspired optimization can enhance neural network performance for a practical financial application. By automating the tedious process of feature engineering and model tuning, the system allows users to focus on interpreting signals rather than configuring algorithms. Additionally, storing predictions in a database creates an audit trail that supports disciplined, evidence-based investment decision-making.

1.4 Objectives

  • To build a system that downloads and processes real-time stock market data from Yahoo Finance.
  • To compute meaningful technical indicators (SMA, EMA, RSI, MACD, Momentum, Volatility) from raw OHLCV data.
  • To implement a Genetic Algorithm for automated feature selection and ANN hyperparameter optimization.
  • To train an MLP classifier that predicts the next-session market direction.
  • To develop a professional, interactive Streamlit-based dashboard with Plotly chart visualizations.
  • To integrate MongoDB Atlas for persistent storage and retrieval of prediction history.

1.5 Scope of the Project

The system supports 50+ tickers from both US (NYSE/NASDAQ) and Indian (NSE) markets. Users can select predefined tickers or enter custom Yahoo Finance symbols. The prediction model works on daily OHLCV data with configurable date ranges, GA population sizes, and generation counts. The application is intended as a decision-support prototype for academic and demonstration purposes, and predictions should not be treated as financial advice.

Chapter 2: Literature Review

2.1 Stock Market Prediction Using Machine Learning

Numerous studies have explored the use of machine learning for stock market prediction. Patel et al. (2015) compared Support Vector Machines, Random Forest, ANN, and Naïve Bayes classifiers for predicting stock trends using ten technical indicators, finding that ANN and Random Forest performed best. Vijh et al. (2020) used ANN models to forecast stock closing prices, achieving strong results on Indian stock market data. These studies established that neural networks can effectively capture non-linear patterns in financial time series.

2.2 Artificial Neural Networks in Finance

Multi-Layer Perceptron (MLP) networks have been widely used for financial forecasting. Kara et al. (2011) employed a feed-forward ANN to predict stock direction on the Istanbul Stock Exchange and demonstrated that ANN outperformed traditional statistical models. The advantage of MLPs lies in their ability to learn complex, non-linear mappings from input features (technical indicators) to output classes (up or down movement) through backpropagation training.

2.3 Genetic Algorithms for Optimization

Genetic Algorithms (GAs), introduced by Holland (1975) and popularized by Goldberg (1989), are metaheuristic optimization techniques that simulate natural evolution. GAs maintain a population of candidate solutions (genomes), apply selection, crossover, and mutation operators, and iteratively evolve toward better solutions. In the context of machine learning, GAs have been successfully used for feature selection, hyperparameter tuning, and neural architecture search.

2.4 Hybrid ANN-GA Approaches

The combination of ANN and GA has been explored by several researchers. Kim and Han (2000) used GA to optimize the connection weights and feature subset of an ANN for stock index prediction. Chien and Chen (2008) applied GA to select features and optimize ANN topology for financial time series forecasting. These hybrid approaches consistently outperformed standalone ANN models, as the GA's global search capability helps avoid local optima in hyperparameter space.

2.5 Technical Indicators

Technical indicators are mathematical transformations of price and volume data that quantify market trends, momentum, and volatility. Commonly used indicators include:

  • Simple Moving Average (SMA) – Average of closing prices over a fixed window.
  • Exponential Moving Average (EMA) – Weighted average that gives more importance to recent prices.
  • Relative Strength Index (RSI) – Measures overbought/oversold conditions on a 0–100 scale.
  • MACD (Moving Average Convergence Divergence) – Captures momentum shifts via EMA crossovers.
  • Volatility – Standard deviation of returns, capturing market risk.
  • Momentum – Rate of price change over a lookback period.

2.6 Gap in Existing Work

While prior studies have demonstrated the viability of hybrid ANN-GA models, most lack a user-friendly deployment platform. Additionally, few systems provide persistent storage for predictions, making it difficult to audit model behavior over time. This project addresses both gaps by integrating the hybrid model into a premium Streamlit dashboard with MongoDB-backed prediction logging.

Chapter 3: Problem Statement

3.1 Problem Definition

Predicting the short-term direction of stock prices is a challenging task due to the stochastic, non-linear, and multi-factorial nature of financial markets. Traditional statistical models fail to capture the complex interactions among technical indicators, and manually tuning machine learning models for different stocks and time periods is time-consuming and error-prone.

3.2 Specific Challenges

  1. Non-linearity: Stock prices are influenced by complex, non-linear relationships between numerous factors. Linear models cannot adequately capture these relationships.
  2. Feature selection: With multiple technical indicators available, selecting the optimal subset of features for prediction is a combinatorial optimization problem.
  3. Hyperparameter sensitivity: The performance of neural networks is highly sensitive to hyperparameters such as hidden layer size, activation function, and learning rate. Manual tuning is impractical across diverse stocks.
  4. Data noise: Financial markets contain significant noise, making it difficult to distinguish genuine signals from random fluctuations.
  5. Lack of audit trails: Most existing prediction tools do not log previous predictions, making it impossible to evaluate model consistency over time.

3.3 Proposed Solution

This project proposes a Stock Market Trend Prediction System that addresses the above challenges through:

  • A Hybrid ANN-GA model where the Genetic Algorithm automatically selects the best feature subset and optimizes ANN hyperparameters, eliminating manual tuning.
  • An MLP Neural Network that captures non-linear relationships between technical indicators and market direction.
  • A professional Streamlit dashboard with interactive charts for visual analysis.
  • A MongoDB Atlas database for persistent storage and retrieval of all prediction runs, creating a searchable audit trail.

Historical stock data is fetched using the yfinance Python library, which provides OHLCV data from Yahoo Finance. The system supports a configurable date range and offers a fallback data source via Alpha Vantage API. Users may also upload offline CSV files in the standard Yahoo Finance format.

4.4 Target Variable

The target variable is a binary label: 1 if the next day's closing price is higher than the current day's (uptrend), and 0 otherwise (downtrend). This is computed as: Target = (Close.shift(-1) > Close).astype(int).

4.5 Genetic Algorithm Design

The Genetic Algorithm performs a simultaneous search over the feature subset and ANN hyperparameters. Each individual in the population (a "genome") encodes:

  • Feature mask: A binary vector of length 11, where each bit indicates whether a feature is included.
  • Hidden layer size: Number of neurons in the hidden layer, chosen from {16, 24, 32, 48, 64}.
  • Activation function: Either relu or tanh.
  • Learning rate: Initial learning rate, chosen from {0.0005, 0.001, 0.003, 0.005}.

4.5.1 GA Operators

  • Selection: Fitness-proportional selection retains the top one-third of the population as survivors.
  • Crossover: Single-point crossover on the feature mask; random selection of hyperparameters from either parent.
  • Mutation: Each feature bit has an 8% chance of flipping; hidden layer size mutates with 15% probability; activation mutates with 10% probability; learning rate mutates with 15% probability.

4.5.2 Fitness Function

The fitness of a genome is computed as the validation accuracy of the MLP trained with the genome's feature subset and hyperparameters, minus a small penalty (5%) proportional to the number of selected features. This encourages parsimonious models: 

fitness = accuracy - (num_selected_features / total_features) × 0.05

4.6 Neural Network Architecture

The system uses a Multi-Layer Perceptron (MLP) Classifier from scikit-learn with:

  • A single hidden layer with GA-optimized neuron count (16–64 neurons)
  • GA-selected activation function (relu or tanh)
  • GA-optimized learning rate (0.0005–0.005)
  • Maximum 700 training iterations for the final model (400 during GA fitness evaluation)
  • Standard scaling applied to input features before training

4.7 Evaluation Metrics

The system computes the following classification metrics on the 20% held-out test set:

  • Accuracy: Proportion of correctly classified samples.
  • Precision: Proportion of predicted uptrends that were actually uptrends.
  • Recall: Proportion of actual uptrends that were correctly predicted.
  • F1 Score: Harmonic mean of precision and recall.
  • Confusion Matrix: 2×2 matrix showing true positives, false positives, true negatives, and false negatives.

5.2 Module Description

5.2.1 Configuration Module (services/config.py)

A dataclass-based configuration loader that reads environment variables for MongoDB connection URI, database name, and Alpha Vantage API key. Uses python-dotenv for loading .env files.

5.2.2 Data Service Module (services/data_service.py)

Responsible for downloading stock data from Yahoo Finance (with 3 retry attempts and Alpha Vantage fallback), computing all 11 technical indicators, and generating market snapshots. Key methods:

  • download_stock_data() – Fetches OHLCV data with caching (TTL: 15 min).
  • build_feature_frame() – Engineers all technical indicators and target labels.
  • current_market_snapshot() – Returns latest OHLCV values and daily change.

5.2.3 Model Service Module (services/model_service.py)

Contains the HybridTrendPredictor class which implements the full ANN-GA pipeline:

  • Genome dataclass – Encodes feature mask and ANN hyperparameters.
  • fit_predict() – Splits data (80/20), runs GA optimization, trains final MLP, returns all metrics.
  • _run_genetic_search() – Evolves population through selection, crossover, and mutation.
  • _fitness() – Evaluates each genome by training a quick MLP and measuring validation accuracy.
  • _random_genome(), _crossover(), _mutate() – GA operators.

5.2.4 Database Service Module (services/db_service.py)

Manages the MongoDB Atlas connection and provides CRUD operations:

  • store_prediction() – Inserts a prediction document with symbol, metrics, genome, and timestamp.
  • fetch_recent_predictions() – Retrieves the latest N prediction records.
  • search_predictions() – Filters by ticker symbol and date range.

5.2.5 Application Module (app.py)

The main Streamlit application (773 lines) that wires all services together and renders five pages:

  1. Landing Page – Hero section with platform overview and quick navigation.
  2. Prediction Dashboard – Candlestick chart, RSI, MACD, probability gauge, stat cards.
  3. Model Insights – Architecture diagram, confusion matrix, accuracy/precision/recall/F1, genome details.
  4. History / Logs – Searchable MongoDB prediction history with filters.
  5. About Project – Technical summary and system information cards.

Chapter 7: Conclusion

7.1 Summary

This project successfully designed and implemented a Stock Market Trend Prediction System using a hybrid Artificial Neural Network–Genetic Algorithm approach. The system demonstrates that evolutionary optimization can effectively automate the challenging tasks of feature selection and hyperparameter tuning for neural network classifiers in the financial domain.

Key accomplishments of this project include:

  • Developed a modular, service-oriented Python application with clearly separated concerns across data fetching, feature engineering, model training, and database operations.
  • Implemented a Genetic Algorithm that simultaneously optimizes feature subsets, hidden layer sizes, activation functions, and learning rates—eliminating the need for manual model configuration.
  • Built a premium, chart-first Streamlit dashboard with interactive Plotly visualizations including candlestick charts, RSI, MACD, probability gauges, and confusion matrices.
  • Integrated MongoDB Atlas for persistent prediction logging, enabling users to search, audit, and compare historical prediction runs across tickers and time periods.
  • Supported 50+ stock tickers from US and Indian markets, with configurable date ranges and GA parameters.

7.2 Limitations

  • The model relies solely on technical indicators and does not incorporate fundamental data, news sentiment, or macroeconomic factors.
  • The GA optimization can be computationally expensive for large population sizes and generation counts.
  • Stock market prediction is inherently uncertain—the system is a decision-support tool and should not be used as the sole basis for investment decisions.
  • The system uses daily OHLCV data and does not support intraday prediction.

7.3 Future Scope

  • Sentiment Analysis: Integrate natural language processing to analyze financial news and social media sentiment as additional input features.
  • Deep Learning Models: Replace the MLP with LSTM or Transformer-based architectures for better temporal pattern recognition.
  • Multi-Step Forecasting: Extend predictions beyond single-day direction to multi-day price trajectory forecasting.
  • Portfolio Optimization: Combine trend predictions with portfolio optimization algorithms for automated asset allocation.
  • Real-Time Streaming: Integrate WebSocket-based real-time data feeds for intraday monitoring and prediction.
  • Ensemble Methods: Combine multiple models (ANN, SVM, Random Forest) with a meta-learner for improved robustness.
  • Mobile Application: Develop a mobile-friendly version or companion app for on-the-go access.
